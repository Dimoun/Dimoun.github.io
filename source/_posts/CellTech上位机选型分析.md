# CellTech 细胞分离设备上位机技术选型分析

## 1.引言

本文档为 CellTech 细胞分离设备的上位机（HMI/控制单元）提供全面的技术选型评估。设备采用“主从式”架构：底层 STM32 主控单元负责毫秒级硬实时控制，上位机负责复杂算法分析、数据管理及人机交互。  
选型范围涵盖 **CPU 架构、操作系统、主板形态、图形界面框架**，并深度融合工控机供货形态（整机/准系统/板卡）与镜像烧录策略，旨在平衡研发效率、系统稳定性与长期运维成本。

---
## 2. 系统架构总览
设备采用分层解耦架构，明确“硬件实时控制”与“业务逻辑处理”的边界。
- **主控层 (STM32)**：精确流体控制、电机驱动、传感器采集、故障保护，具备硬件实时性。
- **显控层 (上位机)**：UI 显示、细胞图像分析、数据存储、外部通信。
- **通信层**：CAN / RS-485 (Modbus RTU)，通过完善协议栈实现解耦。

> **关键约束**：上位机 不承担硬实时任务，所有实时控制由 STM32 完成。因此，本架构对上位机的实时性要求属于软实时范畴。标准 Linux 内核或 Windows 10 IoT 均可完全满足通信周期的确定性需求，无需在上位机引入硬实时操作系统。

# 1. CPU 架构
## 1.1 x86架构
x86架构由英特尔（Intel）于1978年推出，是复杂指令集计算机（CISC）的代表。它以其强大的处理能力和广泛的软件兼容性而闻名。

### 优势
- **性能强大**：在处理复杂任务和运行大型应用程序方面表现出色，尤其在桌面和服务器领域占据主导地位。
- **生态成熟**：拥有庞大且成熟的软件生态系统，支持各种操作系统（如Windows、Linux）和应用程序。
- **兼容性广**：向下兼容性好，能够运行较旧的软件和驱动。

#### 在细胞分离设备中的考量
- **复杂数据处理**: 适用于需要进行大量图像处理、数据分析（如细胞识别、分类算法）以及复杂控制逻辑的细胞分离设备。
- **高性能计算**: 若设备涉及AI辅助分析、高速数据采集与处理，x86能提供所需的计算能力。

### 劣势
- **功耗较高**：通常比ARM架构消耗更多电力，不适合对功耗有严格要求的设备。
- **成本较高**：处理器和相关硬件的成本通常较高。

## 1.2 ARM架构
ARM架构由Acorn Computers于1980年代推出，是精简指令集计算机（RISC）的代表。它以其高能效和低功耗而广泛应用于移动设备和嵌入式系统。

### 优势
- **低功耗**：设计之初就注重能效，非常适合电池供电的移动设备和对功耗敏感的场景。
- **成本较低**：通常具有较低的生产成本，有助于降低整体设备成本。
- **体积小巧**：处理器通常更小，便于集成到紧凑型设备中。

#### 在细胞分离设备中的考量
- **低功耗与紧凑设计**: 适用于对设备尺寸和功耗有严格要求的便携式或集成度高的细胞分离设备。
- **实时控制**: 部分 ARM 处理器集成 Cortex-M 内核，可在芯片层面辅助处理简单、高实时性的控制逻辑，为主控 STM32 分担部分任务。

### 劣势
- **原始性能相对较低**：在单核性能上，通常不如同级别的x86处理器（尽管差距正在缩小）。
- **软件兼容性**：虽然生态系统日益壮大，但在某些传统桌面和服务器应用方面，兼容性仍不如x86。

## 1.3 X86架构与ARM架构对比

| 特性     | x86架构                                    | ARM架构                                        |
| :------- | :----------------------------------------- | :--------------------------------------------- |
| **指令集** | CISC（复杂指令集计算机）                   | RISC（精简指令集计算机）                       |
| **性能**   | 通常更高，尤其在复杂计算和大型应用方面     | 通常较低，但在特定优化场景下表现优异           |
| **功耗**   | 较高，不适合电池供电或对散热有严格要求的设备 | 较低，非常适合移动设备、嵌入式系统和服务器边缘计算 |
| **成本**   | 处理器及硬件成本较高                       | 处理器及硬件成本较低                           |
| **生态系统** | 桌面和服务器领域生态成熟，软件兼容性好     | 移动和嵌入式领域生态成熟，服务器和桌面生态正在发展 |
| **应用场景** | 高性能细胞图像分析、复杂数据处理、大型实验室设备             | 便携式细胞分离设备、实时控制模块、边缘计算单元 |

**决策边界**：
- 需运行 Windows 或商业图像分析 SDK（如 Halcon、Cognex VisionPro、Keyence 等） → **优先选择 x86 架构**
    - 原因：绝大多数商业级机器视觉与图像分析 SDK 仅提供 x86 架构的 Windows 版或 Linux 版，对 ARM 架构的兼容性极差或完全不存在。即便某些 SDK 宣称支持 Linux，往往也只认证特定 x86 发行版（如 Ubuntu x86_64）。若强行在 ARM 平台上运行，会面临：
        - 无可用的二进制安装包或依赖库；
        - 算法加速模块（如 GPU 加速、特定指令集优化）无法调用；
        - 供应商不提供技术支持，导致项目风险不可控。
    - 替代方案：若因成本或功耗原因希望使用 ARM 架构，则需放弃商业 SDK，改用开源图像处理库（如 OpenCV）并自行开发/训练算法模型，这需要额外的计算机视觉团队投入。
- 使用自研 Linux 图像处理、被动散热、内存 ≤ 4 GB → **可考虑 ARM架构**

# 2. 操作系统
## 2.1 Windows 操作系统
Windows是由微软公司开发的一系列操作系统，以其友好的用户界面和广泛的软件兼容性而闻名，尤其在桌面市场占据主导地位。在工业控制领域，Windows通常与HMI/SCADA系统结合使用。

### 优势
- **用户界面友好**：图形用户界面（GUI）直观易用，降低了操作人员的学习曲线。
- **软件生态丰富**：拥有大量的商业软件、驱动程序和开发工具，尤其在工业自动化领域有众多成熟解决方案。
- **硬件兼容性广**：支持多种硬件设备，易于集成。
- **技术支持完善**：微软提供官方支持，且社区资源丰富。

#### 在细胞分离设备中的考量
- **人机交互界面 (HMI)**: 适用于需要复杂、图形化用户界面的细胞分离设备，操作员可以直观地进行参数设置、流程监控和结果查看。
- **与现有实验室系统集成**: 如果实验室已普遍使用Windows平台，可便于与现有数据管理、LIMS等系统集成。
- **商业软件兼容性**: 若某些细胞分析或控制软件仅支持Windows平台，则Windows是必然选择。

### 劣势
- **系统开销大**：相对于Linux，Windows系统资源占用较高，可能需要更高配置的硬件。
- **实时性较差**：标准版Windows并非实时操作系统（RTOS），在对时间确定性有严格要求的工业控制场景可能力不从心。
- **安全性挑战**：作为最普及的操作系统，Windows面临的网络安全威胁较多，需要持续的维护和更新。
- **授权费用**：商业授权通常需要支付费用。

**实时性说明**：若设备上位机无需硬实时，标准 Linux 或 Windows 均可满足通信周期需求；

## 2.2 Linux 操作系统
Linux是一类开源的类Unix操作系统。它以其稳定性、安全性、灵活性和高度可定制性而受到开发者和企业用户的青睐，在服务器、嵌入式系统和工业控制领域应用广泛。

### 优势
- **稳定性高**：长期运行稳定可靠，非常适合需要不间断运行的工业设备。
- **安全性好**：开源特性使得安全漏洞能更快被发现和修复，且其权限管理机制更为严格。
- **系统资源占用低**：内核精简，对硬件要求相对较低，适合资源受限的嵌入式设备。
- **高度可定制**：可以根据具体需求裁剪和优化系统，去除不必要的组件，提升性能和安全性。
- **免费开源**：无授权费用，降低了开发和部署成本。
- **实时性支持**：通过RTAI、PREEMPT_RT等补丁，Linux可以实现接近硬实时的性能，满足工业控制的需求。

**实时性说明**：若设备上位机无需硬实时，标准 Linux 或 Windows 均可满足通信周期需求；

- **镜像与部署**：
  - 必须使用 **主板供应商提供的 BSP** 构建（Yocto / Buildroot），生成含专用驱动的黄金镜像。
  - 量产时可借助 USB 批量烧录或 TFTP 网络部署。

##### 镜像烧录策略详解

###### Windows 工控机镜像方案
- **推荐方式**：使用供应商提供的 Windows 镜像或自行制作定制化镜像
- **工具**：Rufus、WinPE 自动部署工具、DISM++、Macrium Reflect
- **流程**：
  1. 在样机阶段完成全部软件安装、配置与调试。
  2. 使用 ICD 创建包含应用、驱动和系统设置的配置包（.ppkg）。
  3. 启用统一写入筛选器 (UWF)，保护系统分区免受意外断电或写入操作损坏。
  4. 配置自定义 Shell 启动器，使设备开机即全屏运行上位机应用，锁定为专用设备。
  5. 制作“黄金镜像”后，通过 USB 或网络（PXE）批量部署到量产设备。
- **注意事项**：
  - 确保激活正版 Windows 授权（批量授权或 OEM 授权）
  - 制作"黄金镜像"后务必验证每台设备的驱动兼容性

###### Linux 工控机镜像方案
- **推荐方式**：基于供应商 BSP 构建定制 Linux 镜像（Yocto 或 Buildroot），也可自行烧录发行版镜像
- **工具**：Yocto/Buildroot 构建系统、etcher、dd 命令
- **流程**：
  1. 使用 Yocto 构建包含 Linux kernel + 驱动 + 应用的完整镜像
  2. 使用 DD 命令或etcher 烧录到 SSD/eMMC
  3. 量产时通过 USB 批量烧录（推荐）或 TFTP 网络部署
- **黄金镜像原则**：
  - 镜像应包含所有硬件驱动、运行时库、应用二进制
  - 应用层代码可通过 Docker 容器或独立分区部署，镜像更新时保留数据分区
  - 仅使用供应商验证的 BSP 更新包，不随意在线升级内核或驱动

###### ARM 核心板镜像方案
- **烧录工具**：以IMX8举例，NXP提供的 uuu（Universal Update Utility）或 JTAG 调试器
- **流程**：
  1. 使用 Yocto/Buildroot 构建 ARM 专用镜像（含 GPU/VPU 驱动）
  2. 通过 uuu 将镜像烧录到 eMMC
  3. 量产时制作 SD 卡启动镜像，方便工厂快速部署
- **注意事项**：
  - i.MX 8 采用 SPL + U-Boot 分级引导，烧录需遵循供应商指定流程
  - GPU 加速依赖专有固件（Vivante），镜像构建必须包含这些二进制文件

#### 在细胞分离设备中的考量
- **系统稳定性与安全性**: 长期运行且对数据安全要求高的细胞分离设备，Linux的稳定性与安全性优势明显。
- **成本控制**: 对于大规模部署或成本敏感的设备，免费开源的Linux能显著降低授权成本。

### 劣势
- **用户界面相对复杂**：对于不熟悉命令行操作的用户，学习曲线相对陡峭。
- **软件兼容性**：商业软件和驱动程序相对Windows较少，部分特定硬件可能缺乏官方支持。
- **技术支持分散**：开源社区的技术支持，可能不如商业操作系统那样集中和标准化。

## 2.3 Windows 和 Linux 操作系统对比

| 特性       | Windows 操作系统                               | Linux 操作系统                                       |
| :--------- | :--------------------------------------------- | :--------------------------------------------------- |
| **用户界面** | 友好直观，易于上手                             | 相对复杂，对命令行要求较高（但桌面环境如Ubuntu也很友好） |
| **软件生态** | 极其丰富，商业软件和驱动众多                   | 相对较少，开源软件丰富，部分商业软件缺失             |
| **稳定性**   | 良好，但长期运行可能需要重启以维护性能         | 极高，适合长时间不间断运行                           |
| **安全性**   | 面临较多网络威胁，需持续更新和防护             | 良好，开源特性使其漏洞能更快修复，权限管理严格       |
| **系统开销** | 较高，需要较好硬件配置                         | 较低，适合资源受限设备                               |
| **成本**     | 通常需要商业授权费用                           | 免费开源                                             |
| **定制性**   | 有限                                           | 极高，可根据需求裁剪优化                             |
| **技术支持** | 微软官方和大量第三方支持                       | 主要来自社区，部分发行版提供商业支持                 |
| **应用场景** | 复杂HMI、与Windows生态深度集成、商业分析软件依赖的细胞分离设备    | 高稳定性、成本敏感、定制化程度高的细胞分离设备 |
# 3. 主板形态与硬件方案
## 3.1 工控机
工控机（Industrial PC, IPC）是专为工业环境设计和制造的计算机，通常具有高稳定性、可靠性、抗干扰能力强和宽温工作范围等特点。

### 优势
- **环境适应性强**：能在恶劣的工业环境下（如高温、低温、潮湿、多尘、震动、电磁干扰）稳定运行。
- **高可靠性与稳定性**：采用工业级元器件和设计，保证长时间无故障运行。
- **扩展性好**：通常提供丰富的I/O接口和扩展槽，方便连接各种工业设备和模块。
- **标准化**：通常遵循工业标准，便于集成和维护。

#### 在细胞分离设备中的考量
- **环境适应性**: 细胞分离设备可能需要在洁净室或有震动、湿度控制的实验室环境中运行，工控机的环境适应性是关键。
- **I/O扩展**: 需要连接多种传感器（如流量、压力、温度）、执行器（如泵、阀、电机）和光学组件，工控机丰富的I/O接口优势明显。
- **长期稳定运行**: 细胞分离过程可能耗时较长且对稳定性要求极高，工控机的可靠性确保实验过程不中断。

### 劣势
- **成本较高**：由于采用工业级设计和元器件，价格通常高于同等性能的商用电脑。
- **更新换代相对较慢**：为了保证稳定性，技术更新速度可能不如消费级产品。

### 3.1.1 工控机硬件形式

在工业控制与嵌入式应用领域，根据集成度和应用场景，工控硬件通常分为三大类：成品整机、准系统和单板计算机（板卡）。

#### 1. 成品整机 (Ready-to-use System)
这是最完整的硬件形态，相当于一台已经封装好、经过测试的工业 PC。

- **定义：** 厂家将主板、CPU、内存、存储、电源、散热系统全部组装并封装在工业机箱内。
- **优点：**
    - **开箱即用：** 省去了繁琐的组装和调试过程。
    - **可靠性保障：** 成品经过严格的环境耐受性测试（震动、高温、电磁兼容性）。
    - **售后一体化：** 供应商提供整机保修，责任清晰。
- **缺点：** 定制灵活性最低，成本相对较高。
- **适用场景：** 项目周期紧、缺乏硬件集成能力的系统集成项目。
- **镜像**：供应商按要求烧录含 OS + 驱动 + 应用的整体镜像。

#### 2. 准系统 (Barebone)
介于成品和板卡之间，为客户提供了灵活性与便利性的平衡。

- **定义：** 包含机箱、电源、散热系统和核心主板，但**不含 CPU、内存和存储（硬盘/SSD）**。
- **优点：**
    - **配置灵活：** 可根据具体算力需求自行选配 CPU 和存储，成本控制更具弹性。
    - **库存友好：** 方便备货，按需快速配置。
- **缺点：** 需要客户具备一定的硬件装配能力，并需自行处理散热匹配测试。
- **适用场景：** 对计算资源有不同要求、需要快速响应市场的行业方案供应商。
- **镜像**：自购 CPU/内存后，仍需依据主板型号，用供应商 BSP 制作镜像。

#### 3. 核心板卡 / 单板计算机 (SBC - Single Board Computer)
这是工业设备的核心大脑，是集成到更大系统中的基础组件。

- **定义：** 仅包含 PCB 电路板及核心元器件（CPU、芯片组、各类接口），没有外壳和电源。常见的形态包括嵌入式单板机、SOM（核心板+载板）以及 PC/104 总线卡等。
- **优点：**
    - **高度集成：** 可直接嵌入到客户的自有产品外观中。
    - **体积极小：** 适合空间严苛的工业嵌入式场景。
    - **成本优势：** 在大规模量产时具备显著成本优势。
- **缺点：** 研发门槛最高，需要客户具备专业的结构设计、散热设计及电路验证能力。
- **镜像**：需要完整的底层 BSP 适配，镜像通过 uuu 或 JTAG 烧录到 eMMC。

#### 4. 三种形式对比 

| 特性 | 成品整机 | 准系统 | 板卡/模块 (SBC/SOM) |
| :--- | :--- | :--- | :--- |
| **研发门槛** | 低 | 中 | 高 |
| **成本** | 高 | 中 | 低（量大时更优） |
| **定制化程度** | 低 | 中 | 极高 |
| **主要用途** | 自动化项目交付 | 行业方案快速响应 | 嵌入式产品定制开发 |

**选型推荐**：
- 若选 Windows → **仅 x86 工控机（整机或准系统）**
- 若选 Linux → x86 工控机（高性能）或 ARM 核心板（紧凑低功耗）

### 3.1.2 工控机升级维护
工控机的升级维护相对简单，通常遵循PC的维护方式，可更换硬盘、内存条、扩展卡等。但由于其特殊性，建议使用原厂或兼容的工业级备件。

#### 3.1.2.1 硬件升级维护
- **模块化设计**: 许多工控机采用模块化设计，方便更换CPU、内存、存储、I/O模块等。
- **备件供应**: 工业设备生命周期长，确保长期备件供应至关重要。

#### 3.1.2.2 操作系统升级维护
- **定期更新**: 及时打补丁和更新操作系统以修复安全漏洞，但需谨慎测试以确保兼容性。
- **系统备份与恢复**: 定期备份系统镜像，以便快速恢复，减少停机时间。


## 3.2 i.MX 8核心板
i.MX 8系列是NXP（恩智浦）推出的一系列高性能、低功耗的多核应用处理器，基于ARM Cortex-A/M架构。i.MX 8开发板通常指搭载这类处理器的评估或开发套件，它们在多媒体处理、图形渲染、实时控制和边缘AI方面表现出色，广泛应用于工业HMI、医疗影像、智能家居和汽车电子等领域。

### 优势
- **高性能与多核异构**：集成了Cortex-A应用处理器和Cortex-M实时处理器，可实现复杂的应用处理和可靠的实时控制。
- **丰富的多媒体功能**：强大的图形处理单元（GPU）和视频编解码器，支持高清显示和多摄像头输入，适用于图像处理和可视化。
- **低功耗设计**：ARM架构的固有优势，适合对能效有要求的设备。
- **丰富的接口**：提供多种工业标准接口，便于连接传感器、显示器和通信模块。
- **NXP生态支持**：拥有广泛的NXP官方和社区支持，提供操作系统BSP（Board Support Package）和开发工具。

#### 在细胞分离设备中的考量
- **图像与视频处理**: i.MX 8强大的图形和多媒体处理能力，非常适合细胞分离设备中高分辨率图像采集、实时视频分析（如细胞计数、形态识别）和结果的可视化。
- **辅助实时控制与数据处理**: 异构架构允许Cortex-M核处理精确的实时控制任务（如微流控、电机控制），而Cortex-A核处理复杂的上位机逻辑和数据分析。
- **边缘AI**: 支持在设备端进行初步的细胞分类、异常检测等AI推理，减少对云端的依赖，提高响应速度和数据安全性。
- **紧凑型设计**: 适用于对设备体积有严格限制，需要高性能且集成度高的便携式或桌面型细胞分离设备。

### 劣势
- **工业环境适应性**：相较于成品工控机，裸板级的核心板+载板方案在宽温、防尘、防震、抗干扰等工业防护等级上，完全依赖客户的结构与防护设计。
- **开发复杂度高**：多核异构系统、底层 BSP 适配和定制载板设计增加了系统级开发的复杂性，需要专业的嵌入式开发团队。
- **软件生态不如x86成熟**：虽然Linux生态日益完善，但在某些特定的商业软件或Windows独占应用方面，兼容性仍是挑战。

### 3.2.1 i.MX 8核心板操作系统与软件架构
在量产方案中，i.MX 8 通常运行异构软件架构：

- **Cortex-A 核**：运行 NXP 官方长期支持的、基于 **Yocto Project** 构建的定制 Linux 发行版。这是工业应用的首选，可获得最佳稳定性、安全更新和驱动支持。Buildroot 是另一个轻量化选择，适用于资源更受限的场景。

- **Cortex-M 核**：运行 FreeRTOS 或裸机程序，处理高确定性、低延迟的简单任务，与 A 核上的 Linux 通过 RPMsg 等机制通信。

### 3.2.2 i.MX 8 方案升级维护
i.MX 8 方案的系统升级通常通过更新固件（firmware）和操作系统镜像完成。由于采用 SOM 设计，硬件维护高度简化，通常直接更换核心板模块即可完成升级或故障维修。这种模块化设计使其在大规模部署时运维成本很低。

## 3.3 工控机与 i.MX 8 核心板方案对比

| 特性       | 工控机                                       | i.MX 8核心板                                       |
| :--------- | :------------------------------------------- | :------------------------------------------------- |
| **设计目的** | 工业环境长期稳定运行                         | 嵌入式应用、边缘AI、紧凑型多媒体处理                 |
| **环境适应性** | 极强（宽温、防尘、防震、抗干扰）             | 高度依赖客户自研的载板、结构及防护设计）           |
| **可靠性**   | 极高，工业级元器件                           | 高（工业级SoC，但系统可靠性取决于整体方案设计）     |
| **扩展性**   | 强，丰富I/O接口和扩展槽                      | 较强，多GPIO，丰富多媒体及高速接口                 |
| **成本**     | 较高                                         | 中（SoC成本低，但研发投入高；量大时BOM成本优势明显） |

# 4. 图形界面框架

图形用户界面（GUI）框架的选择直接影响开发效率、用户体验和长期维护成本。本章节针对细胞分离设备的 HMI 需求，对比 WPF 和 Qt 两大主流框架。

## 4.1 WPF

WPF（Windows Presentation Foundation）是微软推出的 Windows 桌面应用程序 UI 框架，基于 .NET 平台，利用 DirectX 实现丰富的图形渲染能力。

### 优势
- **强大的图形渲染能力**：基于 DirectX，能够创建视觉效果丰富、动画流畅的用户界面，支持 3D 图形。
- **声明式 UI (XAML)**：使用 XAML 语言定义 UI，将 UI 设计与业务逻辑分离，提高开发效率和可维护性。
- **数据绑定**：强大的数据绑定机制，简化数据与 UI 的同步，减少代码量。
- **样式和模板**：提供高度可定制的 UI 样式和控件模板，方便实现品牌化界面。
- **与 .NET 生态无缝集成**：充分利用 .NET 框架的各种功能和类库。
- **Visual Studio 强大支持**：业界领先的 IDE，调试和开发体验优秀。

### 劣势
- **仅限 Windows 平台**：无法直接跨平台运行。
- **学习曲线**：XAML、数据绑定、依赖属性等概念对初学者较复杂。
- **资源消耗**：相比 WinForms 等传统 UI 框架，启动时内存和 CPU 消耗较高。

### 在细胞分离设备中的考量
- **复杂数据可视化**：适合流式细胞仪数据、显微图像、实时曲线绘制等高质量数据可视化。
- **高定制化 HMI**：能够创建高度定制化、美观的工业界面。
- **Windows 工控机集成**：与 Windows 工控机完美配合，发挥硬件性能。
- **长期维护**：结合 C# 强大生态，便于功能扩展和长期维护。

## 4.2 Qt

Qt 是跨平台的 C++ 图形用户界面应用程序框架，不仅提供丰富的 UI 控件，还包含网络、数据库、多媒体等模块。

### 优势
- **真正的跨平台**：一套代码可编译运行在 Windows、Linux、macOS、Android、iOS 及各种嵌入式系统。
- **性能接近原生**：基于 C++ 开发，编译后程序性能高，资源占用低。
- **功能全面**：除 UI 外，还提供网络通信、数据库访问、多媒体处理等模块。
- **授权灵活**：提供 GPL/LGPL 开源协议和商业授权两种模式。
- **工业级组件**：Qt Charts、Qt DataVisualization 等适合工业数据可视化。

### 劣势
- **学习曲线相对陡峭**：C++ 语言本身学习难度较高。
- **开发效率可能低于 C#**：相较于 C# 快速开发环境，开发周期可能更长。
- **IDE 相对不如 Visual Studio**：Qt Creator 等 IDE 调试功能不如 VS 强大。
- **部署复杂性**：部署时需妥善管理对 Qt 核心库（如 QtCore, QtGui）的依赖。

### 在细胞分离设备中的考量
- **跨平台部署**：若设备需要在多种操作系统上运行，Qt 提供一致体验。
- **高性能图形与实时数据展示**：适合高性能图形渲染、实时曲线、图像显示。
- **底层硬件控制**：基于 C++，易于与底层硬件接口直接通信。

## 4.3 WPF 与 Qt 对比

| 特性         | WPF                                          | Qt                                              |
| :----------- | :------------------------------------------- | :---------------------------------------------- |
| **语言**     | C#（基于 .NET）                              | C++                                             |
| **平台**     | 仅 Windows 平台                              | 真正的跨平台（Windows, Linux, macOS, 嵌入式等） |
| **开发效率** | 高，Visual Studio 强大，组件库丰富           | 相对较低，C++ 语言本身复杂                      |
| **运行性能** | 良好，接近原生                               | 极佳，原生 C++ 性能                             |
| **生态系统** | Windows 桌面应用、企业级                      | 桌面、嵌入式、移动、工业控制                    |
| **学习曲线** | 相对平缓（对 C# 开发者）                     | 较陡峭                                          |
| **资源占用** | 运行时占用相对较高                           | 较低，高效                                      |
| **部署**     | 仅 Windows 平台部署简单                      | 跨平台部署相对复杂                              |
| **授权**     | 微软商业授权，部分开源                        | GPL/LGPL 开源，也可商业授权                     |

**决策边界**：
- Windows 工控机 + C# 团队 + 复杂企业级界面 → **WPF 首选**
- 跨平台部署（Windows/Linux）或 Linux 工控机 → **Qt**

# 5. 结论与选型建议

## 5.1 选型决策矩阵

根据细胞分离设备的具体需求场景，给出如下选型建议：

### 场景 A：高端机型（需要商业视觉 SDK + Windows）

**推荐配置**：
- **CPU 架构**：x86
- **操作系统**：Windows 10 IoT Enterprise 或 Windows 11
- **主板形态**：x86 工控机（准系统，方便按需配置）
- **GUI 框架**：WPF
- **图像处理**：Halcon / Cognex VisionPro / Keyence SDK

**适用机型**：高端细胞分选仪、具备 AI 辅助分析功能的高端型号

**选型理由**：
- 高端机型需要运行商业级机器视觉 SDK，这些 SDK 仅支持 x86 Windows
- 工控机提供强固的工业级可靠性
- WPF 与 .NET 生态无缝集成，适合复杂企业级界面

### 场景 B：中端机型（自研算法 + Linux + 适中成本）

**推荐配置**：
- **CPU 架构**：x86
- **操作系统**：Linux（Ubuntu LTS 或 Debian）
- **主板形态**：x86 工控机（成品整机）
- **GUI 框架**：Qt（LGPL 协议
- **图像处理**：OpenCV + 自研算法

**适用机型**：中端细胞分离设备、注重成本控制的方案

**选型理由**：
- Linux + OpenCV 可显著降低软件授权成本
- x86 工控机性能充裕，生态成熟
- Qt LGPL 协议允许在遵守义务的前提下用于商业应用

### 场景 C：入门机型/紧凑型（ARM + Linux + 低功耗）

**推荐配置**：
- **CPU 架构**：ARM（i.MX 8 系列）
- **操作系统**：Linux（Yocto 构建）
- **主板形态**：i.MX 8 核心板 (SOM) + 定制载板
- **GUI 框架**：Qt（LGPL 协议）
- **图像处理**：OpenCV（ARM 优化版）

**适用机型**：便携式细胞分离设备、紧凑型桌面设备、成本敏感项目

**选型理由**：
- ARM + Linux 方案 BOM 成本最低
- i.MX 8 异构架构可辅助承担部分简单控制任务
- 适合对功耗和体积有严格要求的场景

### 场景 D：快速原型 / 研发阶段

**推荐配置**：
- **CPU 架构**：x86
- **操作系统**：Windows 或 Linux
- **主板形态**：普通 PC / 笔记本电脑（研发阶段）
- **GUI 框架**：WPF（Windows）或 Qt（Linux）
- **图像处理**：OpenCV

**适用场景**：产品研发、算法验证、原型机

**选型理由**：
- 开发资源丰富，调试工具完善
- 可快速迭代功能
- 研发完成后可平移至对应工控机平台

## 5.2 风险与应对

| 风险                          | 可能的选型陷阱                          | 应对策略                                      |
| :---------------------------- | :------------------------------------- | :------------------------------------------- |
